Reference Guide

Workflow

Paste a value from your WAF logs → select match options → click the output or any suggestion to copy the regex to your clipboard → paste into the Block Pattern → Regex field on the Grafana security dashboard → apply the block.

For IP-based blocks, use the Block IP / CIDR / ASN form directly — it's faster and doesn't require regex.

Match Type

Contains — Matches if the value appears anywhere in the target string.
Value: wp-admin → Matches: /wp-admin/setup.php, /blog/wp-admin/, /old-wp-admin-backup
Exact Match — Matches only if the entire target string equals the value. Nothing before, nothing after.
Value: /wp-admin/ → Matches: /wp-admin/ only. Does NOT match: /wp-admin/setup.php
Starts With — Matches if the target string begins with the value.
Value: /wp- → Matches: /wp-admin/, /wp-content/, /wp-includes/. Does NOT match: /blog/wp-admin
Ends With — Matches if the target string ends with the value.
Value: .php → Matches: /setup.php, /xmlrpc.php. Does NOT match: /setup.php?id=1

Case Sensitivity

Case Insensitive (default) — Matches regardless of upper/lower case. Recommended for most blocking rules since attackers often randomize case to evade detection.
Value: admin → Matches: admin, Admin, ADMIN, aDmIn
Case Sensitive — Only matches the exact casing provided. Use when you need precision and the value is always a specific case (like a file extension or protocol header).
Value: POST → Matches: POST only. Does NOT match: post, Post

Encoding Detection

Attackers frequently URL-encode characters to bypass WAF rules that only match literal strings. These options generate regex that catches both the literal value and its encoded forms.

URL Encoded (%XX) — Matches the literal value AND its standard URL-encoded form. Each character is matched as either itself or its %HexCode equivalent.
Value: ../ → Also matches: %2e%2e%2f, %2E%2E/, .%2e/ (any mix of encoded and literal characters)
Value: /etc/passwd → Also matches: %2fetc%2fpasswd
Nested/Double Encoded (%25XX) — Catches double-encoding evasion, where the % character itself is encoded as %25. Some WAFs decode URL encoding once — attackers double-encode to survive that first decode pass.
Value: ../ → Also matches: %252e%252e%252f (double-encoded), %2e%2e/ (single-encoded), ../ (literal)
This is the most thorough option — it covers literal, single-encoded, AND double-encoded forms simultaneously.

Additional Options

Negate (match everything EXCEPT this) — Creates a negative lookahead that matches any request that does NOT contain the value. Useful for positive security: "block everything except this known-good pattern."
Value: /philosophy/ with Negate → Blocks any request whose path does NOT contain /philosophy/
Value: Mozilla with Negate → Blocks any request whose User-Agent does NOT contain Mozilla (catches non-browser bots)
Wildcard Segments — Replaces path separators with flexible wildcards (/.*?/), allowing any content between path segments. Useful for blocking a pattern that appears in different directory depths.
Value: /wp/admin/setup.php → Regex: /wp/.*?/admin/.*?/setup\.php
This matches: /wp/admin/setup.php, /wp/v2/admin/panel/setup.php, /wp/anything/admin/here/setup.php
Word Boundaries — Wraps the value in \b boundary markers so it only matches as a complete word, not as part of a longer string. Prevents over-blocking.
Value: admin → Matches: admin, /admin/, admin panel. Does NOT match: administrator, sysadmin, admintools
Value: test → Matches: test, test-page. Does NOT match: testing, latest, contest
IP Address Mode — Automatically escapes dots (. → \.) which is required for IP addresses in regex. Without this, 1.2.3.4 would match 102030400 because . means "any character" in regex.
Value: 192.168.1.100 → Regex: 192\.168\.1\.100
Note: For simple IP blocks, the Block IP form on the dashboard is faster — it doesn't require regex at all. Use this option when you need to match IPs within a larger pattern.

Tips

  • Options combine — you can select "Starts With" + "Case Insensitive" + "URL Encoded" simultaneously.
  • Green toggles = encoding options active. Orange toggles = additional options active.
  • Smart Suggestions below the output are independent of your toggle selections — they always show common useful patterns for the value you entered.
  • Click any green output text (main output or suggestions) to copy to clipboard instantly.
  • When blocking paths, prefer Starts With over Contains to avoid accidentally blocking legitimate paths that share substrings.
  • When blocking user-agents, prefer Contains + Case Insensitive since UA strings vary wildly in casing and format.
